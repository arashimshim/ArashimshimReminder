# 要件定義・設計書：iOS/iPadOS 音声リマインダーアプリ (キッズ・プログレス)

## 1. プロジェクト概要
* **目的**: 小学4年生の子供が、自立してスケジュールや持ち物を管理できるよう、リマインダーの内容を「音声」で読み上げて通知し、日々のタスク達成をポイント制で楽しく継続できるiPad専用アプリ。
* **ターゲット層**: 小学4年生（9〜10歳前後）の子供、およびその保護者（タスク管理・設定・レポート確認用）。
* **主要な差別化ポイント**: 
    * 画面を見なくても、iPadが声（音声合成）で「宿題の時間だよ」「明日の準備をしよう」と教えてくれる。
    * **「開始時刻」だけでなく「完了時刻」も管理し、やっていない（未完了）場合に自動で「追いかけ声かけ」をして行動を促す。**
    * **ゲーミフィケーション要素**: タスク完了時、および「1日のタスク全完了時」に音声と画面で最大級の褒め言葉（「よくできたね！」など）を贈る。
    * **ポイント・レポートシステム**: 1日の全タスクをコンプリートするとポイントが加算され、週単位で集計・振り返りができるレポート機能を搭載。
* **開発環境**: iPadOS 17以上, Swift, SwiftUI, SwiftData (ローカルDB), AVFoundation (読み上げ用)

## 2. 要件定義 (Requirements Definition)

### 2.1 機能要件 (Functional Requirements)
* **タスク（リマインダー）管理機能**
    * 「やること（タイトル）」の登録・編集・削除。
    * **時間設定**: 「開始時刻」と「完了時刻（目安）」の2つを登録可能。
    * **曜日別設定**: 月〜日の曜日ごとにタスクを登録・更新できる（複数曜日の一括選択も可能）。

* **音声通知（読み上げ・追いかけ声かけ）機能 【最重要】**
    * **開始時の通知**: 開始時刻になったら、タスクのタイトルや設定された文章を音声で読み上げる。
    * **追いかけ声かけ（未完了時）**: 開始時刻を過ぎてもタスクが「完了」にチェックされていない場合、一定間隔（例: 5分おき）で「まだ終わっていないよ、がんばろう！」などと音声で催促（声かけ）を行う。
    * 子供が聞き取りやすいよう、音声の「速度」「ピッチ（高さ）」を調整可能。

* **モチベーション向上・ゲーミフィケーション機能**
    * **個別タスク完了時の褒めアクション**: タスク完了（おわった！ボタンタップ）時に、効果音とともに「よくできたね！」「その調子！」といったランダムな褒め言葉を音声と画面で演出。
    * **1日全完了（コンプリート）ボーナス**: その日のすべてのタスクを完了した瞬間、特別なファンファーレ音とともに「今日のやること全部クリア！本当に素晴らしい、よくがんばったね！」と最大級の褒めコメントを音声再生。
    * **ポイント加算**: 1日の全タスクを完全に終わらせられた場合、当日分の「コンプリートポイント（例: 100pt）」を加算。

* **週次レポート・ポイント集計機能**
    * 獲得したポイントを週単位（月曜日〜日曜日）で自動集計。
    * 過去の週、および今週の獲得ポイント、タスク完了率を可視化するレポート画面の提供。

### 2.2 非機能要件 (Non-Functional Requirements)
* **データ永続化**: 曜日ごとのタスクデータ、日々の完了実績、およびポイント履歴をSwiftDataでローカルに保存。
* **完全スタンドアロン**: アカウント登録やサーバー通信は一切不要。子供の利用状況や個人情報を完全にiPad内に閉じ込めることで安全性を担保。
* **日付変更時の自動リセット処理**: 日付が変わったタイミング（または毎朝4時など）で、当日のタスク完了フラグを自動リセットし、前日のコンプリート判定を確定させてポイント履歴を生成する。

## 3. 画面設計 & UI/UX (Screen Design)

### 3.1 画面一覧 (iPad向け 3ペイン/タブ構成を想定)
1.  **メイン画面（子供用・曜日別タスク一覧）**:
    * 画面上部に「月」「火」「水」…と曜日タブがあり、現在の曜日のページがデフォルトで開く。
    * タスクが大きな文字と視覚的なカード形式で一覧表示される（開始〜完了時間、ステータス）。
    * 各タスクカードに、手動で音声を鳴らせる「きく」ボタン、そして大きな「おわった！」ボタン。
    * **コンプリート演出**: 全タスクが完了すると、画面全体に紙吹雪のバナーが表示され、褒めコメントが表示される。
2.  **レポート画面（子供・保護者用・ポイント確認）**:
    * **今週の進捗**: 今週の合計獲得ポイントと、曜日ごとのクリア状況（スタンプや星マークなど）を子供向けに分かりやすく表示。
    * **過去のレポート**: 週単位のポイント推移、タスク達成率をグラフやシンプルなリストで表示し、保護者が子供の頑張りを振り返り・評価できるようにする。
3.  **タスク登録・編集画面（保護者用）**:
    * やること（タイトル）、開始時刻、完了時刻、適用する曜日（複数選択可能）を設定。
4.  **アプリ設定画面**:
    * 音声の種類（話者の変更）、音量、速度テスト。追いかけ声かけの間隔（5分/10分/15分）の設定。

## 4. システム設計 (System Design)

### 4.1 アーキテクチャ
* **パターン**: MVVM (Model-View-ViewModel) + SwiftUI
* **通知・音声制御**: `UNUserNotificationCenter`によるローカル通知と、`AVSpeechSynthesizer`を用いた`SpeechManager`（シングルトン）による音声制御。

### 4.2 データモデル設計 (Data Model)

#### `WeeklyTask` (曜日別のマスタータスク設定)
| プロパティ名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | `UUID` | 一意の識別子 |
| `title` | `String` | やること（例:「宿題」「明日のじゅんび」） |
| `targetDays` | `[Int]` | 適用する曜日の配列 (1:日, 2:月, ..., 7:土) |
| `startTime` | `Date` | 開始時刻 (時・分のみ保持) |
| `endTime` | `Date` | 完了時刻 (時・分のみ保持) |

#### `DailyTaskLog` (日々の個別タスク実施ログ)
| プロパティ名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | `UUID` | 一意の識別子 |
| `taskId` | `UUID` | `WeeklyTask`のIDへの参照 |
| `date` | `Date` | 実施日（日付のみ） |
| `isCompleted` | `Bool` | 完了フラグ |
| `completedAt` | `Date?` | 実際に完了ボタンを押した日時 |

#### `DailyHistoryLog` (1日単位のコンプリート状況・ポイント履歴)
| プロパティ名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | `UUID` | 一意の識別子 |
| `date` | `Date` | 記録対象の日付 |
| `isFullyCompleted`| `Bool` | その日のタスクをすべて完了したか |
| `pointsEarned` | `Int` | 獲得ポイント（全完了なら100、未達成なら0など） |

## 5. 実装フェーズへの指示 (AIへのプロンプト用バッファ)
* **ポイント・レポート集計のロジック**:
    * 週の集計は、Swiftの `Calendar` を使用して `component(.weekOfYear, from: Date)` もしくは指定した月〜日の期間で `DailyHistoryLog` をフィルタリング・グルーピングしてポイントを合計すること。
* **コンプリート判定のリアルタイム制御**:
    * ユーザーがタスクの「おわった！」ボタンを押すたびに、その日の該当曜日に設定されている全タスク数と、`DailyTaskLog` の完了数を比較し、一致した瞬間に即座にコンプリート音声の再生と画面の演出（紙吹雪など）を実行すること。
* **音声再生の割り込み防止**:
    * 褒め言葉の音声再生中に次の通知が重なった場合など、音声が不自然に途切れないよう、`AVSpeechSynthesizerDelegate` を用いたキュー管理、または再生中フラグの制御を行うこと。

## 6. テスト仕様・シナリオ (Test Specifications)

AIで単体テスト（Unit Test）コードを生成する場合、および手動で動作確認を行うためのテスト項目です。

### 2.1 単体テスト（データ・ロジック検証）項目
* **[UT-001] 曜日別判定ロジックの検証**
    * **検証内容**: ターゲット曜日（例：月・水）に設定されたタスクが、当日の曜日と一致する場合のみ一覧に抽出されること。
    * **期待値**: カレンダーが「月曜日」の時はタスクAが表示され、「火曜日」の時は非表示になること。
* **[UT-002] 1日コンプリート（全完了）判定の検証**
    * **検証内容**: その日の該当曜日に登録されているすべての `WeeklyTask` に対して、`DailyTaskLog` の `isCompleted` が true になった瞬間にコンプリートと判定されること。
    * **期待値**: タスクが3つある場合、3つ目を完了した瞬間に `DailyHistoryLog` に100ptが加算され、全完了フラグが true になること。タスクが0個の曜日はコンプリート扱いにならないこと。
* **[UT-003] 週次ポイント集計ロジックの検証**
    * **検証内容**: 過去7日間（月〜日）の `DailyHistoryLog` の `pointsEarned` を集計し、正しい合計値が算出されること。
    * **期待値**: 週に3回コンプリート（100pt×3）していた場合、週合計が「300pt」とレポートに表示されること。
* **[UT-004] 日付変更時の自動リセット検証**
    * **検証内容**: 日付が変わった際、前日の完了フラグがマスターデータ（`WeeklyTask`）に影響を与えず、新しい日の `DailyTaskLog` が未完了（false）の状態でクリーンに生成されること。

### 6.2 統合・UIテスト（音声・挙動確認）シナリオ
* **[IT-001] 開始時刻の音声通知シナリオ**
    * **手順**: 
        1. テストタスク（例：「宿題」、開始15:00、完了16:00）を当日の曜日に設定。
        2. iPadのシステム時間を14:59に設定し、アプリをバックグラウンド（またはスリープ）にする。
        3. 15:00になるまで待つ。
    * **期待値**: 15:00になった瞬間にローカル通知が届き、アプリ起動中であれば「宿題の時間だよ」と設定された音声がクリアに再生されること。
* **[IT-002] 未完了時の「追いかけ声かけ」シナリオ**
    * **手順**: 
        1. [IT-001] の状態（15:00に開始）から、タスクを「おわった！」にせず放置する。
        2. 設定された追いかけ間隔（例：5分）が経過するまで待つ（15:05）。
    * **期待値**: 15:05に、2回目の催促音声（「まだ終わっていないよ、がんばろう！」など）が自動でトリガーされること。
* **[IT-003] タスク完了時の割り込み防止シナリオ**
    * **手順**: 
        1. 追いかけ声かけの音声（または開始音声）が流れている最中に、画面の「おわった！」ボタンをタップする。
    * **期待値**: 喋っていた催促の音声が即座に停止、または自然にフェードアウトし、上書きで「よくできたね！」という褒め言葉の音声と完了効果音が優先して再生されること（音声が重なって不協和音にならないこと）。
